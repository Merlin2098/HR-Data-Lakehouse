# ADR-003: Externalized ETL Assets in S3

- Short Name: `externalized-etl-assets`
- Status: Implemented

## Context

Glue jobs require Python entrypoints, SQL files, and YAML configuration. These assets must be versioned and deployed consistently across environments.

## Decision

Store ETL scripts, SQL, and configuration files in a dedicated S3 `scripts` bucket, with Git as the source of truth and Terraform handling deployment.

## Alternatives Considered

- Embedding scripts directly in Glue job definitions
  - Simpler to start
  - Harder to version, review, and reuse

## Consequences

### Positive

- Clear separation between infrastructure and ETL logic
- Better versioning and traceability
- Reusable assets across local and AWS execution modes

### Negative

- Additional deployment surface
- More artifact references to manage

## Architecture Impact

Introduces an explicit artifact layer that separates data storage from executable logic.
