# ADR-006: Parquet BI Snapshot Delivery

- Short Name: `parquet-bi-snapshot`
- Status: Implemented

## Context

The project needs a low-friction way to hand off curated data to desktop BI tools without depending on Athena drivers, managed BI services, or live connectivity.

## Decision

Export a stable single-file CSV snapshot from gold for local BI consumption.

## Consequences

### Positive

- Simple delivery path
- No runtime dependency on ODBC/JDBC drivers
- Works well with local desktop analytics tools

### Negative

- Not real-time
- Snapshot usage remains manual after export

## Architecture Impact

Introduces a BI handoff stage that is intentionally decoupled from live query connectivity.
