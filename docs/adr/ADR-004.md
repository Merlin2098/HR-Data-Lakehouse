# ADR-004: Incremental Terraform Delivery

- Short Name: `incremental-terraform-delivery`
- Status: Evolving

## Context

The project started as a functional demo and needed a fast path to working infrastructure without over-investing in enterprise-grade delivery on day one.

## Decision

Use Terraform as the deployment mechanism with a manual-first workflow, while keeping the repository ready to evolve toward CI/CD automation.

## Alternatives Considered

- Full production-grade CI/CD from the beginning
  - More enterprise-ready
  - Overkill for the current MVP and demo scope

## Consequences

### Positive

- Faster implementation
- Direct control during iterative architecture changes

### Negative

- More operator-driven deployment steps
- Operational maturity remains partial

## Architecture Impact

Defines a clear path from demo-ready infrastructure toward more automated environment promotion later.
