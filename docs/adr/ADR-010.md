# ADR-010: Cost Monitoring Baseline

- Short Name: `cost-monitoring-baseline`
- Status: Implemented

## Context

The project aims to simulate a production-oriented cloud setup and needs basic visibility into spending during demos and iterative development.

## Decision

Implement AWS Budgets and consistent tagging as the initial cost-monitoring baseline.

## Consequences

### Positive

- Better financial visibility
- Stronger operational discipline

### Negative

- Adds a small amount of infrastructure and alerting complexity

## Architecture Impact

Cost awareness becomes part of the infrastructure baseline instead of an afterthought.
