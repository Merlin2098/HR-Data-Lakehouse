# ADR-008: Parquet as the Core Analytical Format

- Short Name: `parquet-core-format`
- Status: Implemented

## Context

The project needs efficient storage and query performance for Athena and downstream analytical consumption.

## Decision

Use Parquet as the main format for silver, gold, and the BI snapshot.

## Consequences

### Positive

- Columnar storage for efficient scans
- Lower Athena scan costs
- Better compression than CSV

### Negative

- Higher implementation complexity than raw text formats

## Architecture Impact

Parquet becomes the standard format for all curated layers and exported analytical outputs.
