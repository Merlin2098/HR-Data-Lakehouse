# ADR-001: Event-Driven Landing Trigger

- Short Name: `event-driven-landing-trigger`
- Status: Implemented

## Context

The pipeline processes HR attrition CSV files with an expected low batch frequency. The project needs a simple, decoupled, and serverless way to start processing only when a new landing file appears.

## Decision

Use `S3 Object Created` events filtered through EventBridge to start the Step Functions state machine.

## Alternatives Considered

- Direct S3-to-Lambda trigger
  - Simpler at first
  - Less flexible for multi-stage orchestration
- Scheduled execution with EventBridge Scheduler
  - Time-based control
  - Does not react to real data arrival

## Consequences

### Positive

- Event-driven architecture
- Loose coupling between storage and orchestration
- Fully serverless trigger path

### Negative

- Dependence on the S3 event payload shape
- Requires payload normalization downstream

## Architecture Impact

The landing zone becomes the single pipeline entry point and can support future ingestion sources without changing the ETL stages.
