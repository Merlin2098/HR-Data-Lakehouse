# ADR-011: Terraform as Infrastructure as Code

- Short Name: `terraform-iac`
- Status: Implemented

## Context

The project needs versioned, portable, and reviewable infrastructure definitions across environments.

## Decision

Use Terraform instead of CloudFormation as the IaC tool.

The Terraform stack is also intentionally configured for easy demo lifecycle management:

- Athena workgroups are recursively deletable
- S3 buckets use destroy-friendly settings even when they contain versioned objects and delete markers
- operator commands are expected to use environment-specific `tfvars` for `plan`, `apply`, and `destroy`

## Alternatives Considered

- CloudFormation
  - Native to AWS
  - Less portable and less aligned with the project's preferred workflow

## Consequences

### Positive

- Better portability
- Strong fit with CI/CD evolution
- Flexible debugging and modularization
- Easier end-to-end demo teardown without manual AWS cleanup

### Negative

- Extra tooling layer to learn and maintain
- `destroy` is intentionally powerful across all environments and must be used carefully

## Architecture Impact

Infrastructure becomes cloud-managed through modular IaC with clear environment separation.
The infrastructure contract also includes explicit lifecycle behavior for deploy and teardown, not only resource creation.
