# ADR-005: Tableau Public as an Optional Publication Path

- Short Name: `tableau-public-optional`
- Status: Deferred

## Context

The project explored low-cost ways to publish analytics without adopting a paid managed BI service.

## Decision

Do not make Tableau Public part of the active runtime path. Keep it only as an optional future publication channel for public-facing demos.

## Alternatives Considered

- QuickSight as the primary BI path
  - Native AWS integration
  - Additional service cost
- Tableau Public as the active runtime target
  - No extra AWS-side service cost
  - Public sharing model is not appropriate for the current private workflow

## Consequences

### Positive

- Avoids coupling the core pipeline to a public publishing platform
- Keeps BI publication choices open

### Negative

- Public dashboard publishing is not automated
- Tableau remains a manual, optional downstream consumer

## Architecture Impact

BI publication is intentionally separated from the active ETL runtime.
