# ADR-009: Step Functions as the Orchestrator

- Short Name: `step-functions-orchestrator`
- Status: Implemented

## Context

The ETL requires ordered execution, explicit control flow, retry handling, and visibility across multiple Glue stages.

## Decision

Use AWS Step Functions as the orchestrator for the ETL pipeline.

## Consequences

### Positive

- Clear workflow visibility
- Better failure handling and retries
- Explicit state transitions between ETL stages

### Negative

- Additional orchestration complexity

## Architecture Impact

Orchestration becomes a first-class part of the runtime rather than implicit chaining between jobs.
